# Discussion

Our preliminary experiments suggest that pure zero-shot LLM-based machine sound anomaly detection is not yet reliable in this setting. When the model is asked to judge a target clip without any matched normal reference, performance is unstable across both DSP-only and image-based inputs. In contrast, once a small set of normal reference clips from the same case is provided, performance improves substantially and becomes much more consistent. This indicates that the present task is better framed as reference-based anomaly detection rather than absolute zero-shot health assessment. In other words, the models appear much more capable of answering "does this clip deviate from this machine's normal behavior?" than "does this clip sound globally abnormal in isolation?"

Reference construction turned out to be a central design choice. Restricting the support set to the same case was necessary, but channel selection also mattered more than we initially expected. Same-channel references consistently outperformed mixed-channel references, and statistical analysis showed why: mixing channels substantially broadened the normal reference distribution. By comparison, increasing the number of same-channel references from 4 to 16 only modestly changed the baseline statistics, which helps explain why performance improved from 4-shot to 16-shot but then saturated. Taken together, these results suggest that reference quality matters more than reference quantity. A relatively small but distributionally matched support set is more useful than a larger but acoustically heterogeneous one.

The way reference information is presented to the model is also important. Explicitly providing multiple raw normal reference examples did not outperform aggregated statistical summaries. This is a notable negative result, because it suggests that more context is not automatically beneficial for LLM-based anomaly detection. In our setting, the models performed better when the support set was compressed into baseline summary statistics than when they were required to infer the normal pattern directly from multiple examples. This observation is consistent with the broader trend in our experiments: information organization appears to matter at least as much as information volume.

Feature analysis further clarified why some prompts over-predicted anomalies. The DSP features are not independent; several of them form highly correlated groups, especially spectral brightness features (`spectral_centroid`, `high_frequency_ratio`, `spectral_rolloff`) and energy/mel-related features (`rms_energy`, `log_mel_mean`, `log_mel_std`). This redundancy appears to encourage evidence double-counting. In the strongest few-shot settings, the model often treated multiple correlated shifts as if they were independent pieces of abnormal evidence, which increased false positives on acoustically active but still normal clips. This interpretation is supported by the error analysis: the remaining hard false positives were typically normal samples with stronger high-frequency content, broader spectra, or larger short-term fluctuations than their baselines.

An important finding is that feature usefulness depends on the baseline regime. The leave-one-feature-out studies showed that the same feature may be helpful in a low-reference setting and harmful in a high-reference setting. The clearest example was `rms_energy`: it appeared useful at 4-shot, where the baseline estimate is still coarse, but detrimental at 16-shot, where the normal reference is already stable and energy differences are more likely to reflect benign operating variation. This regime dependence suggests that feature selection for LLM-based anomaly detection should not be treated as model-agnostic or context-free. Instead, the feature set should be calibrated jointly with the amount and type of reference information.

Model comparisons led to a similar conclusion. The best-performing configuration was not universal across model families. For `gpt-5.4-mini`, removing `rms_energy` produced the best evaluation performance. For `gemini-3.1-flash-lite-preview`, performance improved further when the thinking level was increased to `medium`, and the best variant removed `rms_energy` together with one additional correlated feature such as `log_mel_std`. These differences imply that prompt and feature calibration are model-specific. At the same time, the hardest errors were shared across OpenAI and Gemini systems, suggesting that some remaining failures are primarily data-representation problems rather than simple model-capability limitations.

The held-out test results indicate that some evaluation-set overfitting is present, but not enough to overturn the main conclusion. The tuned Gemini configuration reached substantially higher performance on the evaluation split than on the held-out test split, which shows that the later rounds of feature selection and model-specific tuning were influenced by the evaluation set. However, the held-out test ROC-AUC remained high, and the overall F1 score was still strong enough to support the central claim that reference-based few-shot prompting is viable for this task. The more conservative interpretation, therefore, is not that the method has solved machine sound anomaly detection, but that it demonstrates a promising and analyzable early-stage approach whose main strengths and failure modes are already visible.

Finally, our controlled experiment with a single anomaly type (`ab02`) suggests an important future direction. Adding one or two abnormal exemplars to the prompt substantially improved performance in that restricted setting. We do not yet treat this as a main result because the experiment was conducted on a simplified single-anomaly scenario, while the full ToyConveyor anomalies are combinatorial condition mixtures rather than clean single-cause labels. Nevertheless, the result suggests that exemplar-augmented reference-based prompting may be a powerful extension of the normal-reference setup, especially when the anomaly family is partially known.

Overall, these findings support a practical takeaway: LLM-based machine sound anomaly detection currently looks most promising not as a pure zero-shot problem, but as a reference-based comparison problem with carefully chosen support sets, compact statistical summaries, and model-specific feature calibration. This framing is both more empirically effective and more interpretable than the stronger zero-shot formulation we initially considered.
